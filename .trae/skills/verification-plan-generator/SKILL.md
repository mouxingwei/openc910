# 验证方案生成

## 功能概述

本SKILL根据测试点文档生成芯片验证方案。

## 使用场景

- 测试点文档已完成，需要生成验证方案
- 需要定义验证环境和策略
- 需要规划验证进度和覆盖率目标

## 工作流程

### Step 1: 收集输入

1. 读取测试点文档
2. 确认芯片的功能模块
3. 了解验证资源限制

### Step 2: 分析测试点

从测试点中提取验证需求：

**验证规模**：
- 测试点数量
- 功能模块数量
- 复杂度评估

**验证要求**：
- 覆盖率要求
- 仿真时间要求
- 资源需求

## 验证方法说明

### 直接测试

基于测试点直接编写测试用例：
- 测试点明确、边界清晰的功能
- 需要精确控制激励的场景
- 占比约40%

### 随机测试

使用随机约束生成测试激励：
- 大量边界组合
- 随机场景覆盖
- 占比约40%

### 回归测试

持续运行已有测试用例：
- 确保新修改不破坏已有功能
- 自动化回归流程
- 占比约20%

## 示例

### 输入（测试点）

```
测试点：
- TP-001: 正常中断响应
- TP-002: 中断嵌套
- TP-010: 最大中断数测试
- TP-020: 中断禁用时触发中断
```

## 注意事项

1. **目标明确**：验证目标要具体可衡量
2. **策略合理**：验证方法比例要合理
