# 测试点文档生成

## 功能概述

本SKILL根据明确的需求列表生成芯片测试点文档。

## 使用场景

- 需求分析完成，需要生成测试点文档
- 需要将需求转化为可验证的测试点
- 需要覆盖正常场景、异常场景和边界条件

## 工作流程

### Step 1: 收集输入

1. 读取明确的需求列表（来自requirements-analyzer SKILL的输出）
2. 读取功能规格说明书
3. 确认需要测试的功能模块

### Step 2: 分析需求提取测试点

从需求中提取测试点：

**功能测试点**：
- 正常功能操作
- 各种功能模式
- 功能之间的交互

**边界测试点**：
- 最大值/最小值
- 边界条件
- 临界值

**异常测试点**：
- 异常情况处理
- 错误输入处理
- 超时处理

**性能测试点**：
- 响应时间
- 吞吐量
- 延迟

### Step 3: 生成测试点文档

按以下结构生成文档：

### Step 4: 测试点评审检查

**完整性检查**：
- [ ] 每个功能需求都有对应测试点
- [ ] 边界条件都有覆盖
- [ ] 异常情况都有测试

**可验证性检查**：
- [ ] 测试点描述清晰
- [ ] 测试步骤明确
- [ ] 预期结果可判定

### Step 5: 输出文档

输出Markdown格式测试点文档，可选择生成Word格式。

## 测试点提取规则

### 功能测试点提取

| 需求模式 | 测试点类型 | 示例 |
|----------|------------|------|
| WHEN-THEN | 基本功能 | 正常触发条件下的功能 |
| 多种条件 | 条件分支 | 各条件分支的功能 |
| 状态转换 | 状态功能 | 各状态下的功能 |

### 边界测试点提取

| 边界类型 | 测试点 | 示例 |
|----------|--------|------|
| 数值边界 | 最大/最小值 | 计数器到最大值 |
| 数组边界 | 索引边界 | 数组首尾元素 |
| 时序边界 | 超时边界 | 超时时间到 |

### 异常测试点提取

| 异常类型 | 测试点 | 示例 |
|----------|--------|------|
| 输入错误 | 无效输入 | 非法数据 |
| 时序错误 | 超时 | 响应超时 |
| 资源错误 | 资源耗尽 | 缓冲区满 |

## 示例

### 输入（明确需求）

```gherkin
## Requirement: 中断处理
### Scenario: 外部中断响应
- **WHEN** 外部中断信号有效且中断使能打开
- **THEN** 响应中断并跳转到中断处理程序
- **AND** 保存当前PC到mepc寄存器
```

### 输出（测试点）

```markdown
## 2. 功能测试点

### 2.1 中断控制器功能测试

#### TP-001: 正常中断响应

- **描述**: 验证外部中断能够被正确响应
- **功能模块**: ICU（中断控制器）
- **前置条件**: 
  - 中断使能寄存器(ICU_INT_EN)已配置
  - 全局中断使能(mstatus.MIE)已打开
- **测试步骤**: 
  1. 配置外设中断源
  2. 触发外部中断
  3. 观察处理器响应
- **预期结果**: 
  - 跳转到中断向量地址
  - mepc寄存器保存中断发生时的PC值
  - mcause寄存器正确记录中断原因
- **优先级**: 高
- **测试类型**: 功能
- **关联需求**: REQ-ICU-001

#### TP-002: 中断嵌套

- **描述**: 验证高优先级中断可以打断低优先级中断
- **功能模块**: ICU
- **前置条件**: 已配置中断优先级
- **测试步骤**: 
  1. 触发低优先级中断
  2. 在中断处理中触发高优先级中断
  3. 观察中断嵌套行为
- **预期结果**: 
  - 正确保存低优先级中断上下文
  - 跳转到高优先级中断处理
  - 正确返回低优先级中断继续执行
- **优先级**: 中
- **测试类型**: 功能
- **关联需求**: REQ-ICU-001

---

## 3. 边界测试点

### 3.1 中断边界测试

#### TP-010: 最大中断数测试

- **描述**: 验证系统能够处理所有配置的中断源
- **测试边界值**: 
  - 最大中断数: 64
- **测试步骤**: 
  1. 配置64个中断源
  2. 依次触发每个中断
  3. 验证每个中断都能被正确处理
- **预期结果**: 所有64个中断都能被正确识别和处理
- **优先级**: 中
- **测试类型**: 边界
- **关联需求**: REQ-ICU-002

---

## 4. 异常测试点

### 4.1 中断异常测试

#### TP-020: 中断禁用时触发中断

- **描述**: 验证中断禁用时中断不会得到响应
- **异常条件**: 全局中断禁用(mstatus.MIE=0)
- **测试步骤**: 
  1. 关闭全局中断使能
  2. 触发外部中断
  3. 观察处理器行为
- **预期结果**: 
  - 处理器不响应中断
  - 中断挂起标志被记录
  - 后续开启中断后能够响应
- **优先级**: 高
- **测试类型**: 异常
- **关联需求**: REQ-ICU-003
```

## 注意事项

1. **覆盖完整**：确保每个需求都有对应测试点
2. **优先级合理**：根据功能重要性分配测试优先级
3. **可执行**：测试点要可执行和可验证
4. **可追踪**：测试点要与需求关联
