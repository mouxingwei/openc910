# OpenC910 一级模块文档：硬件辅助调试 (HAD)

## 1. 模块概述

硬件辅助调试 (Hardware Assisted Debug, HAD) 是 OpenC910 处理器的调试支持系统，提供完整的调试功能包括 JTAG 接口、断点、单步执行和程序跟踪。HAD 支持标准 RISC-V 调试规范，可配合 GDB 等调试工具使用。

### 1.1 主要功能

- JTAG 调试接口
- 指令断点
- 数据断点
- 单步执行
- 寄存器访问
- 内存访问
- 程序跟踪

## 5. 调试模式

### 5.1 调试模式进入

| 触发条件 | 描述 |
|---------|------|
| JTAG 请求 | 通过 JTAG 发送调试请求 |
| 指令断点 | PC 匹配断点地址 |
| 数据断点 | 访存地址匹配断点 |
| 单步模式 | 执行单条指令后进入 |

### 5.2 调试模式操作

```
调试模式下可执行:
1. 读写 GPR 寄存器
2. 读写 CSR 寄存器
3. 访问内存
4. 执行单条指令
5. 设置/清除断点
```

## 6. 断点类型

### 6.1 指令断点

| 特性 | 描述 |
|-----|------|
| 数量 | 8-16个 |
| 触发条件 | PC 匹配 |
| 动作 | 进入调试模式 |

### 6.2 数据断点

| 特性 | 描述 |
|-----|------|
| 数量 | 4-8个 |
| 触发条件 | 地址/数据匹配 |
| 访问类型 | Load/Store |
| 动作 | 进入调试模式 |

## 7. 跟踪功能

### 7.1 跟踪类型

| 类型 | 描述 |
|-----|------|
| PC 跟踪 | 记录程序执行路径 |
| 数据跟踪 | 记录访存操作 |
| 指令跟踪 | 记录执行的指令 |

### 7.2 跟踪缓冲

- 循环缓冲存储
- 可配置触发条件
- 支持导出到外部

## 8. JTAG 命令

### 8.1 常用命令

| 命令 | 描述 |
|-----|------|
| AC_ACCESS_REG | 访问寄存器 |
| AC_ACCESS_MEM | 访问内存 |
| AC_ABSTRACT_CMD | 抽象命令 |
| AC_PROGRAM_BUF | 程序缓冲操作 |

### 8.2 DMI 寄存器

| 地址 | 名称 | 描述 |
|-----|------|------|
| 0x04 | dmcontrol | 调试模块控制 |
| 0x05 | dmstatus | 调试模块状态 |
| 0x10 | haltsum0 | 停止核心摘要 |
| 0x17 | abstractcs | 抽象命令状态 |
| 0x18 | command | 命令寄存器 |
| 0x19 | abstractauto | 自动抽象 |

## 9. 设计要点

1. **标准兼容**: 遵循 RISC-V 调试规范
2. **非侵入式**: 调试不影响正常执行
3. **实时性**: 支持实时调试
4. **低功耗**: 调试模块可独立时钟门控
