# OpenC910 二级模块文档：TLB (Translation Lookaside Buffer)

## 1. 模块概述

TLB (Translation Lookaside Buffer) 是 MMU 中用于缓存虚拟地址到物理地址翻译结果的硬件结构。OpenC910 采用两级 TLB 结构：指令 TLB (ITLB)、数据 TLB (DTLB) 和联合 TLB (JTLB)。

### 1.1 所属关系

```
MMU (一级模块)
├── TLB (二级模块) ← 本文档
│   ├── ITLB (指令TLB)
│   ├── DTLB (数据TLB)
│   └── JTLB (联合TLB)
├── 页表遍历
└── 系统映射
```

## 2. TLB 类型

### 2.1 TLB 配置

| TLB 类型 | 容量 | 相联度 | 用途 |
|---------|------|-------|------|
| ITLB | 16项 | 全相联 | 指令地址翻译 |
| DTLB | 16项 | 全相联 | 数据地址翻译 |
| JTLB | 256项 | 4路组相联 | 共享后备TLB |

## 4. TLB 表项结构

### 4.1 表项字段

| 字段 | 位宽 | 描述 |
|-----|------|------|
| VPN | 27 | 虚拟页号 |
| PPN | 44 | 物理页号 |
| ASID | 16 | 地址空间ID |
| G | 1 | 全局映射标志 |
| V | 1 | 有效位 |
| R | 1 | 可读权限 |
| W | 1 | 可写权限 |
| X | 1 | 可执行权限 |
| D | 1 | 脏位 |
| A | 1 | 访问位 |

### 4.2 表项布局

```
┌─────────────────────────────────────────────────────────────┐
│                      TLB Entry                              │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│    VPN      │    PPN      │    ASID     │       G         │
│  (27 bits)  │  (44 bits)  │  (16 bits)  │    (1 bit)      │
├─────────────┼─────────────┼─────────────┼─────────────────┤
│     V       │     R       │     W       │       X         │
│  (1 bit)    │  (1 bit)    │  (1 bit)    │    (1 bit)      │
├─────────────┼─────────────┼─────────────┼─────────────────┤
│     D       │     A       │         Reserved              │
│  (1 bit)    │  (1 bit)    │                               │
└─────────────┴─────────────┴───────────────────────────────┘
```

## 5. TLB 操作

### 5.1 查找 (Lookup)

```
查找流程:
1. 输入虚拟地址 (VA)
2. 提取 VPN
3. 并行比较所有表项的 VPN 和 ASID
4. 匹配成功:
   - 返回 PPN 和权限位
   - 更新访问位
5. 匹配失败:
   - 触发 TLB 缺失
   - 查询下一级 TLB 或触发页表遍历
```

### 5.2 填充 (Fill)

```
填充时机: 页表遍历完成
填充流程:
1. 选择替换表项 (PLRU 算法)
2. 写入新的翻译信息
3. 设置有效位
```

### 5.3 无效化 (Invalidate)

```
无效化类型:
1. SFENCE.VMA: 无效化指定 ASID 的表项
2. 全局无效化: 无效化所有表项
3. 单项无效化: 无效化指定地址的表项
```

## 6. TLB 层次查找

### 6.1 查找流程

```
地址翻译流程:
1. ITLB/DTLB 查找
   ├── 命中: 返回物理地址
   └── 未命中: 继续
2. JTLB 查找
   ├── 命中: 填充微TLB，返回物理地址
   └── 未命中: 继续
3. 触发页表遍历
   └── 获取翻译结果，填充 TLB
```

### 6.2 查找延迟

| TLB | 命中延迟 | 未命中处理 |
|-----|---------|-----------|
| ITLB/DTLB | 1 周期 | 查询 JTLB |
| JTLB | 2-3 周期 | 触发页表遍历 |

## 7. 大页支持

### 7.1 支持的页大小

| 页大小 | VPN 位数 | 描述 |
|-------|---------|------|
| 4KB | 27 | 标准页 |
| 2MB | 18 | 大页 |
| 1GB | 9 | 超大页 |

### 7.2 大页处理

```
大页识别:
1. 检查页表项的叶子标志
2. 根据级别确定页大小
3. 使用对应的 VPN 位数进行匹配
```

## 9. 设计要点

1. **两级结构**: 微TLB + 联合TLB 平衡延迟和容量
2. **大页支持**: 支持 4KB/2MB/1GB 页大小
3. **ASID 支持**: 支持多地址空间
4. **快速查找**: 单周期微TLB命中
