# OpenC910 二级模块文档：返回地址栈 (RAS)

## 1. 模块概述

返回地址栈 (Return Address Stack, RAS) 是 IFU 中专门用于预测函数返回地址的预测器。由于函数返回的目标地址在编译时无法确定，RAS 通过维护一个调用栈来预测返回地址。

### 1.1 所属关系

```
IFU (一级模块)
├── 分支预测子系统
│   ├── L0-BTB (二级模块)
│   ├── BTB (二级模块)
│   ├── BHT (二级模块)
│   └── RAS (二级模块) ← 本文档
```

## 2. 模块列表

| 模块名称 | 文件 | 功能描述 |
|---------|------|---------|
| ct_ifu_ras | ct_ifu_ras.v | RAS 顶层模块 |

## 3. 配置参数

| 参数 | 值 |
|-----|-----|
| 深度 | 8-16 项 |
| 宽度 | 40 位 (PC宽度) |
| 访问延迟 | 1 周期 |

## 4. 工作原理

### 4.1 基本操作

```
CALL 指令:
  - Push: 将返回地址压入栈顶
  - 返回地址 = PC + 指令长度

RET 指令:
  - Pop: 从栈顶弹出预测的返回地址
  - 使用弹出的地址作为预测目标
```

### 4.2 栈结构

```
┌─────────────────┐ ◄── 栈顶 (TOS)
│   Entry[0]      │     返回地址0
├─────────────────┤
│   Entry[1]      │     返回地址1
├─────────────────┤
│   Entry[2]      │     返回地址2
├─────────────────┤
│      ...        │
├─────────────────┤
│   Entry[N-1]    │ ◄── 栈底
└─────────────────┘
```

## 5. 操作流程

### 5.1 Push 操作 (CALL)

```
1. 检测到 CALL 指令
2. 计算返回地址 (PC + 指令长度)
3. 栈指针 +1
4. 将返回地址写入栈顶
```

### 5.2 Pop 操作 (RET)

```
1. 检测到 RET 指令
2. 读取栈顶的返回地址
3. 栈指针 -1
4. 使用读取的地址作为预测目标
```

## 6. 接口信号

### 6.1 预测接口

| 信号名称 | 方向 | 位宽 | 功能描述 |
|---------|------|------|---------|
| ras_push_en | I | 1 | Push 使能 (CALL) |
| ras_push_addr | I | 40 | Push 地址 |
| ras_pop_en | I | 1 | Pop 使能 (RET) |
| ras_pop_addr | O | 40 | Pop 返回地址 |
| ras_valid | O | 1 | RAS 有效 (非空) |

### 6.2 修复接口

| 信号名称 | 方向 | 位宽 | 功能描述 |
|---------|------|------|---------|
| ras_recover_en | I | 1 | 恢复使能 |
| ras_recover_ptr | I | 4 | 恢复指针 |

## 7. 错误恢复

### 7.1 预测错误处理

当分支预测错误时，需要恢复 RAS 状态:

```
1. 保存预测时的栈指针
2. 预测错误发生时
3. 恢复栈指针到预测前的值
4. 重新开始预测
```

### 7.2 栈溢出/下溢处理

| 情况 | 处理方式 |
|-----|---------|
| 栈满时 Push | 丢弃最旧的项，或覆盖 |
| 栈空时 Pop | 返回默认地址，或禁用预测 |

## 8. 与其他预测器的协作

```
分支预测流程:
1. 预解码识别指令类型
2. CALL 指令:
   - BTB 记录目标
   - RAS Push 返回地址
3. RET 指令:
   - 优先使用 RAS 预测
   - RAS 失败时使用 BTB
```

## 9. 设计要点

1. **专门优化**: 专门针对函数返回预测
2. **低延迟**: 单周期访问
3. **错误恢复**: 支持预测错误后的状态恢复
4. **深度可配置**: 可根据应用调整栈深度
