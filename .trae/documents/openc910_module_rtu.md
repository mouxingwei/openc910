# OpenC910 一级模块文档：提交单元 (RTU)

## 1. 模块概述

提交单元 (Retire Unit, RTU) 是 OpenC910 处理器的指令提交控制中心，负责管理指令的顺序提交、异常处理和架构状态更新。RTU 通过重排序缓冲区 (ROB) 实现乱序执行后的顺序提交，确保精确异常和正确的程序语义。

### 1.1 主要功能

- 指令顺序提交
- 重排序缓冲区管理
- 异常检测与处理
- 架构状态更新
- 调试支持
- 分支预测更新

## 5. ROB 结构

### 5.1 ROB 表项内容

| 字段 | 位宽 | 描述 |
|-----|------|------|
| IID | 7 | 指令ID |
| PC | 40 | 程序计数器 |
| Preg | 6 | 目的物理寄存器 |
| Valid | 1 | 表项有效 |
| Complete | 1 | 执行完成 |
| Exception | 5 | 异常向量 |
| ExptValid | 1 | 异常有效 |
| BranchInfo | - | 分支预测信息 |

### 5.2 ROB 操作

```
创建 (Create):
  IDU 发射指令时分配 ROB 表项
  写入指令信息 (PC, Preg等)

完成 (Complete):
  执行单元完成指令执行
  设置完成标志
  记录异常信息

提交 (Retire):
  按顺序检查队首指令
  如果完成且无异常，提交
  更新架构状态
  释放物理寄存器
```

## 6. 提交机制

### 6.1 提交条件

1. 指令位于 ROB 队首
2. 指令执行完成
3. 无更早的异常
4. 无调试停止请求

### 6.2 提交带宽

- 每周期最多提交 3 条指令
- 支持批量提交

### 6.3 提交操作

```
提交时执行:
1. 更新架构寄存器映射
2. 释放旧物理寄存器
3. 更新 PC
4. 发送提交信息到 HPCP
5. 更新分支预测器
```

## 7. 异常处理

### 7.1 异常类型

| 异常 | 向量 | 描述 |
|-----|------|------|
| 指令地址错误 | 0 | 取指地址不对齐 |
| 指令访问错误 | 1 | 取指权限错误 |
| 非法指令 | 2 | 未定义指令 |
| 断点 | 3 | EBREAK 指令 |
| Load地址错误 | 4 | Load地址不对齐 |
| Load访问错误 | 5 | Load权限错误 |
| Store地址错误 | 6 | Store地址不对齐 |
| Store访问错误 | 7 | Store权限错误 |
| 环境调用 | 8 | ECALL 指令 |
| 指令页错误 | 12 | 取指页错误 |
| Load页错误 | 13 | Load页错误 |
| Store页错误 | 15 | Store页错误 |

### 7.2 异常处理流程

```
1. 执行单元检测异常
2. 写入 ROB 表项
3. 等待指令到达队首
4. 刷新流水线
5. 更新 CSR 寄存器
6. 跳转到异常处理程序
```

## 8. 调试支持

### 8.1 调试模式进入

- JTAG 请求
- 断点命中
- 单步模式

### 8.2 调试信息输出

- 提交 PC
- 提交指令信息
- 异常信息

## 9. 设计要点

1. **精确异常**: 确保异常发生时架构状态精确
2. **顺序提交**: 保证程序语义正确
3. **高吞吐量**: 支持每周期3条指令提交
4. **调试友好**: 完整的调试状态输出
