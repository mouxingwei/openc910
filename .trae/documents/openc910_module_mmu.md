# OpenC910 一级模块文档：内存管理单元 (MMU)

## 1. 模块概述

内存管理单元 (Memory Management Unit, MMU) 是 OpenC910 处理器的虚拟内存管理核心，负责虚拟地址到物理地址的翻译。MMU 包含多级 TLB 和页表遍历逻辑，支持 SV39/SV48 虚拟内存模式。

### 1.1 主要功能

- 虚拟地址翻译
- TLB 管理
- 页表遍历
- 内存属性检查
- 地址空间隔离

## 5. TLB 层次结构

### 5.1 TLB 配置

| TLB | 容量 | 相联度 | 用途 |
|-----|------|-------|------|
| ITLB | 16项 | 全相联 | 指令地址翻译 |
| DTLB | 16项 | 全相联 | 数据地址翻译 |
| JTLB | 256项 | 4路组相联 | 共享后备TLB |

### 5.2 TLB 查找流程

```
1. 检查 ITLB/DTLB (微TLB)
   ├── 命中: 返回物理地址
   └── 未命中: 查找 JTLB
       ├── 命中: 填充微TLB，返回物理地址
       └── 未命中: 触发页表遍历
```

## 6. 地址翻译

### 6.1 SV39 地址格式

```
虚拟地址 (39位有效):
┌─────────┬─────────┬─────────┬──────────┐
│ VPN[2]  │ VPN[1]  │ VPN[0]  │  Offset  │
│  9位    │  9位    │  9位    │  12位    │
└─────────┴─────────┴─────────┴──────────┘

物理地址 (56位):
┌─────────┬─────────┬─────────┬──────────┐
│ PPN[2]  │ PPN[1]  │ PPN[0]  │  Offset  │
│ 26位    │  9位    │  9位    │  12位    │
└─────────┴─────────┴─────────┴──────────┘
```

### 6.2 页表项格式

| 字段 | 位 | 描述 |
|-----|-----|------|
| V | 0 | 有效位 |
| R | 1 | 可读 |
| W | 2 | 可写 |
| X | 3 | 可执行 |
| U | 4 | 用户态可访问 |
| G | 5 | 全局映射 |
| A | 6 | 访问位 |
| D | 7 | 脏位 |
| RSW | 8-9 | 保留给软件 |
| PPN | 10-53 | 物理页号 |

## 7. 页表遍历

### 7.1 SV39 遍历流程

```
Level 2: VPN[2] -> 页目录项
    │
    ▼
Level 1: VPN[1] -> 页目录项
    │
    ▼
Level 0: VPN[0] -> 页表项 -> 物理地址
```

### 7.2 大页支持

| 页大小 | 翻译级别 |
|-------|---------|
| 4KB | 3级翻译 |
| 2MB | 2级翻译 (叶子在Level 1) |
| 1GB | 1级翻译 (叶子在Level 2) |

## 8. 内存属性

### 8.1 权限检查

| 检查项 | 描述 |
|-------|------|
| R/W/X | 读/写/执行权限 |

### 8.2 内存类型

| 类型 | 描述 |
|-----|------|
| Cacheable | 可缓存 |
| Bufferable | 可缓冲 |
| Shareable | 可共享 |

## 9. 设计要点

1. **两级TLB**: 微TLB + 联合TLB 提高命中率
2. **硬件页表遍历**: 自动处理TLB缺失
3. **大页支持**: 支持2MB/1GB大页
4. **ASID支持**: 支持地址空间标识符
