# OpenC910 二级模块文档：重排序缓冲区 (ROB)

## 1. 模块概述

重排序缓冲区 (Reorder Buffer, ROB) 是 RTU 的核心组件，用于实现乱序执行后的顺序提交。ROB 按程序顺序存储正在执行的指令信息，确保精确异常和正确的程序语义。

### 1.1 所属关系

```
RTU (一级模块)
├── ROB (二级模块) ← 本文档
├── 物理寄存器状态表
└── 提交控制
```

## 4. ROB 表项结构

### 4.1 表项字段

| 字段 | 位宽 | 描述 |
|-----|------|------|
| valid | 1 | 表项有效 |
| iid | 7 | 指令 ID |
| pc | 40 | 程序计数器 |
| inst | 32 | 指令编码 |
| preg | 6 | 目的物理寄存器 |
| areg | 5 | 目的架构寄存器 |
| complete | 1 | 执行完成标志 |
| expt_vld | 1 | 异常有效 |
| expt_vec | 5 | 异常向量 |
| expt_code | 32 | 异常代码 |
| br_pred | 1 | 分支预测信息 |
| br_mispred | 1 | 预测错误标志 |

### 4.2 表项布局

```
┌─────────────────────────────────────────────────────────────┐
│                     ROB Entry                               │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│   Valid     │    IID      │     PC      │   Inst          │
│   (1 bit)   │  (7 bits)   │ (40 bits)   │  (32 bits)      │
├─────────────┼─────────────┼─────────────┼─────────────────┤
│   Preg      │   Areg      │  Complete   │   Expt Info     │
│  (6 bits)   │  (5 bits)   │  (1 bit)    │   (38 bits)     │
├─────────────┼─────────────┼─────────────┴─────────────────┤
│  Br Info    │           Reserved                          │
│  (2 bits)   │                                             │
└─────────────┴─────────────────────────────────────────────┘
```

## 5. ROB 操作

### 5.1 创建 (Create)

```
创建时机: IDU 发射指令时
创建流程:
1. 分配空闲表项
2. 写入指令信息 (PC, Inst, Preg等)
3. 设置 valid = 1, complete = 0
4. 返回 IID 给执行单元
```

### 5.2 完成 (Complete)

```
完成时机: 执行单元完成指令执行时
完成流程:
1. 执行单元发送完成信号和 IID
2. 根据 IID 定位 ROB 表项
3. 设置 complete = 1
4. 记录异常信息 (如有)
```

### 5.3 提交 (Retire)

```
提交时机: 指令到达队首且完成时
提交流程:
1. 检查队首指令是否完成
2. 检查是否有异常
   - 有异常: 触发异常处理
   - 无异常: 提交指令
3. 更新架构寄存器映射
4. 释放旧物理寄存器
5. 清除表项 (valid = 0)
```

## 6. 指针管理

### 6.1 指针定义

| 指针 | 描述 |
|-----|------|
| Head | 队首指针，指向最老的指令 |
| Tail | 队尾指针，指向下一个分配位置 |

### 6.2 指针操作

```
创建: Tail = (Tail + 创建数) % ROB_SIZE
提交: Head = (Head + 提交数) % ROB_SIZE
刷新: Tail = Head (清空 ROB)
```

## 8. 异常处理

### 8.1 异常记录

```
异常发生时:
1. 执行单元发送异常信息
2. ROB 记录异常到对应表项
3. 设置 expt_vld = 1
```

### 8.2 异常触发

```
异常触发条件:
1. 异常指令到达队首
2. 指令已完成

异常处理:
1. 刷新流水线
2. 保存异常信息到 CSR
3. 跳转到异常处理程序
```

## 9. 设计要点

1. **顺序提交**: 确保程序语义正确
2. **精确异常**: 异常发生时架构状态精确
3. **高吞吐量**: 支持每周期多条指令提交
4. **低延迟**: 快速的创建和完成路径
