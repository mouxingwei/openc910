# OpenC910 二级模块文档：发射队列 (Issue Queue)

## 1. 模块概述

发射队列 (Issue Queue) 是 IDU 中负责指令调度的核心组件，存储等待发射的指令，监控操作数就绪状态，并在条件满足时发射指令到执行单元。

### 1.1 所属关系

```
IDU (一级模块)
├── 发射队列 (二级模块) ← 本文档
│   ├── AIQ0 (ALU队列0)
│   ├── AIQ1 (ALU队列1)
│   ├── BIQ (分支队列)
│   ├── LSIQ (Load/Store队列)
│   ├── SDIQ (特殊指令队列)
│   └── VIQ (向量队列)
├── 寄存器重命名
└── 寄存器文件
```

## 2. 队列类型

### 2.1 队列配置

| 队列名称 | 深度 | 服务单元 | 指令类型 |

## 4. 表项结构

### 4.1 通用表项字段

| 字段 | 位宽 | 描述 |
|-----|------|------|
| valid | 1 | 表项有效 |
| ready | 1 | 指令就绪 |
| src0_preg | 6 | 源操作数0物理寄存器 |
| src0_rdy | 1 | 源操作数0就绪 |
| src1_preg | 6 | 源操作数1物理寄存器 |
| src1_rdy | 1 | 源操作数1就绪 |
| dst_preg | 6 | 目的物理寄存器 |
| iid | 7 | 指令 ID |
| func | 4 | 功能码 |
| imm | 20 | 立即数 |

### 4.2 表项布局


## 5. 操作流程

### 5.1 入队 (Dispatch)

```
入队时机: IDU 重命名阶段完成
入队流程:
1. 选择目标队列 (根据指令类型)
2. 分配空闲表项
3. 写入指令信息
4. 初始化就绪状态 (检查操作数是否就绪)
```

### 5.2 唤醒 (Wakeup)

```
唤醒时机: 执行单元完成指令时
唤醒流程:
1. 执行单元广播完成的物理寄存器
2. 队列比较所有表项的源操作数
3. 匹配的源操作数设置为就绪
4. 如果所有源操作数就绪，设置指令就绪
```

### 5.3 发射 (Issue)

```
发射时机: 指令就绪且执行单元空闲
发射流程:
1. 从就绪指令中选择最老的指令
2. 发送发射请求到执行单元
3. 读取寄存器文件获取操作数
4. 清除表项
```

## 6. 发射仲裁

### 6.1 仲裁策略

```
选择策略:
1. 优先级: 年龄优先 (最老优先)
2. 等待时间: 长时间等待优先
3. 公平性: 防止饥饿
```

### 6.2 仲裁逻辑

```
仲裁流程:
1. 收集所有就绪指令
2. 按年龄排序
3. 选择最老的指令
4. 检查执行单元可用性
5. 发射选中的指令
```

## 8. 设计要点

1. **乱序发射**: 支持指令乱序发射
2. **快速唤醒**: 单周期唤醒逻辑
3. **公平调度**: 防止指令饥饿
4. **高吞吐量**: 支持每周期多条指令发射
