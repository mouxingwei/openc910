# OpenC910 二级模块文档：分支目标缓冲区 (BTB)

## 1. 模块概述

分支目标缓冲区 (Branch Target Buffer, BTB) 是 IFU 中分支预测器的核心组件，用于存储分支指令的目标地址。OpenC910 采用两级 BTB 结构：L0-BTB 和主 BTB，以实现快速且准确的分支预测。

### 1.1 所属关系

```
IFU (一级模块)
├── 分支预测子系统
│   ├── L0-BTB (二级模块)
│   ├── BTB (二级模块) ← 本文档
│   ├── BHT (二级模块)
│   └── RAS (二级模块)
```

## 2. L0-BTB 模块

### 2.1 功能描述

L0-BTB 是小容量、低延迟的分支目标缓冲区，用于在取指阶段快速预测分支目标。

### 2.2 模块列表

| 模块名称 | 文件 | 功能描述 |
|---------|------|---------|
| ct_ifu_l0_btb | ct_ifu_l0_btb.v | L0-BTB 顶层 |
| ct_ifu_l0_btb_entry | ct_ifu_l0_btb_entry.v | L0-BTB 表项 |

### 2.3 配置参数

| 参数 | 值 |
|-----|-----|
| 容量 | 16-32 项 |
| 相联度 | 全相联 |
| 访问延迟 | 1 周期 |

### 2.4 表项结构

| 字段 | 位宽 | 描述 |
|-----|------|------|
| Tag | 10-20 | 地址标签 |
| Target | 40 | 分支目标地址 |
| Valid | 1 | 有效位 |
| Type | 2 | 分支类型 |

## 3. 主 BTB 模块

### 3.1 功能描述

主 BTB 是大容量的分支目标缓冲区，存储更多的分支目标信息，支持更复杂的分支预测。

### 3.2 模块列表

| 模块名称 | 文件 | 功能描述 |
|---------|------|---------|
| ct_ifu_btb | ct_ifu_btb.v | BTB 顶层 |
| ct_ifu_btb_tag_array | ct_ifu_btb_tag_array.v | BTB 标签阵列 |
| ct_ifu_btb_data_array | ct_ifu_btb_data_array.v | BTB 数据阵列 |

### 3.3 配置参数

| 参数 | 值 |
|-----|-----|
| 容量 | 512-1024 项 |
| 相联度 | 4-way 组相联 |
| 访问延迟 | 2 周期 |

### 3.4 存储阵列结构

```
标签阵列:
┌─────────────────────────────────────────┐
│  Tag[0]  │  Tag[1]  │  Tag[2]  │  Tag[3] │  (每组4路)
└─────────────────────────────────────────┘

数据阵列:
┌─────────────────────────────────────────┐
│ Target[0]│ Target[1]│ Target[2]│ Target[3]│
│ Type[0]  │ Type[1]  │ Type[2]  │ Type[3]  │
└─────────────────────────────────────────┘
```

## 4. 间接 BTB 模块

### 4.1 功能描述

间接 BTB 用于预测间接跳转指令的目标地址，这些指令的目标地址在运行时才能确定。

### 4.2 模块列表

| 模块名称 | 文件 | 功能描述 |
|---------|------|---------|
| ct_ifu_ind_btb | ct_ifu_ind_btb.v | 间接 BTB 顶层 |
| ct_ifu_ind_btb_array | ct_ifu_ind_btb_array.v | 间接 BTB 阵列 |

### 4.3 配置参数

| 参数 | 值 |
|-----|-----|
| 容量 | 64-128 项 |
| 相联度 | 2-way 组相联 |

## 5. BTB 操作流程

### 5.1 查询流程

```
1. PC 输入 BTB
2. L0-BTB 并行查询 (1周期)
   ├── 命中: 返回预测目标
   └── 未命中: 查询主 BTB
3. 主 BTB 查询 (2周期)
   ├── 命中: 返回预测目标，填充 L0-BTB
   └── 未命中: 无预测
```

### 5.2 更新流程

```
1. 分支指令执行完成
2. 计算实际目标地址
3. 与预测目标比较
   ├── 匹配: 更新 BHT (如需要)
   └── 不匹配: 更新 BTB，触发预测错误
4. 分配新表项 (如需要)
```

## 6. 替换策略

### 6.1 L0-BTB 替换

- FIFO 替换策略
- 最近使用优先保留

### 6.2 主 BTB 替换

- PLRU (伪 LRU) 替换策略
- 每组维护替换状态

## 7. 接口信号

### 7.1 查询接口

| 信号名称 | 方向 | 位宽 | 功能描述 |
|---------|------|------|---------|
| btb_pc | I | 40 | 查询 PC |
| btb_en | I | 1 | 查询使能 |
| btb_target | O | 40 | 预测目标 |
| btb_hit | O | 1 | 命中标志 |
| btb_type | O | 2 | 分支类型 |

### 7.2 更新接口

| 信号名称 | 方向 | 位宽 | 功能描述 |
|---------|------|------|---------|
| btb_update_en | I | 1 | 更新使能 |
| btb_update_pc | I | 40 | 更新 PC |
| btb_update_target | I | 40 | 更新目标 |
| btb_update_type | I | 2 | 更新类型 |

## 8. 设计要点

1. **两级结构**: L0-BTB + 主 BTB 平衡延迟和容量
2. **快速访问**: L0-BTB 单周期延迟
3. **间接跳转**: 专门的间接 BTB 处理
4. **高效替换**: PLRU 替换策略
