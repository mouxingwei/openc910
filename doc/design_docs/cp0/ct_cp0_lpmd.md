# ct_cp0_lpmd 模块详细设计方案

## 1. 模块概述

### 1.1 模块名称
`ct_cp0_lpmd` - 低功耗管理模块

### 1.2 功能描述
`ct_cp0_lpmd` 是CP0的低功耗管理模块，负责处理器的低功耗模式控制。该模块管理电源状态、中断唤醒以及时钟门控控制，支持多种低功耗模式以降低处理器功耗。

### 1.3 主要特性
- 低功耗模式控制
- 中断唤醒机制
- 电源状态管理
- 时钟门控控制
- 功耗状态寄存器管理

## 2. 低功耗模式

### 2.1 功耗模式定义

| 模式 | 编码 | 描述 |
|------|------|------|
| Normal | 2'b00 | 正常运行模式 |
| Low-Power | 2'b01 | 低功耗模式 |
| Standby | 2'b10 | 待机模式 |
| Sleep | 2'b11 | 睡眠模式 |

### 2.2 模式特性

| 模式 | 时钟 | 功耗 | 唤醒时间 |
|------|------|------|----------|
| Normal | 全开 | 最高 | - |
| Low-Power | 部分门控 | 中等 | 短 |
| Standby | 大部分门控 | 低 | 中 |
| Sleep | 全部门控 | 最低 | 长 |

## 5. 关键逻辑设计

### 5.1 功耗模式状态机

```verilog
// 状态定义
parameter NORMAL   = 2'b00;
parameter LOW_PWR  = 2'b01;
parameter STANDBY  = 2'b10;
parameter SLEEP    = 2'b11;

// 状态转移
always @(posedge clk or negedge cpurst_b) begin
    if (!cpurst_b)
        power_mode <= NORMAL;
    else case (power_mode)
        NORMAL:   if (enter_lp) power_mode <= LOW_PWR;
        LOW_PWR:  if (enter_stdby) power_mode <= STANDBY;
                  else if (wakeup) power_mode <= NORMAL;
        STANDBY:  if (enter_sleep) power_mode <= SLEEP;
                  else if (wakeup) power_mode <= NORMAL;
        SLEEP:    if (wakeup) power_mode <= NORMAL;
    endcase
end
```

### 5.2 中断唤醒逻辑

```verilog
// 中断唤醒条件
// 任何有效的中断都可以唤醒处理器
assign wakeup_req = regs_lpmd_int_vld && 
                    (|regs_iui_int_sel[14:0]);

// 唤醒请求输出
assign lpmd_xx_wakeup_req = wakeup_req;
```

### 5.3 时钟控制

```verilog
// 时钟使能控制
// 根据功耗模式控制时钟使能
always @(*) begin
    case (power_mode)
        NORMAL:   clk_en = 1'b1;
        LOW_PWR:  clk_en = 1'b1;  // 部分门控
        STANDBY:  clk_en = 1'b0;  // 大部分门控
        SLEEP:    clk_en = 1'b0;  // 全部门控
        default:  clk_en = 1'b1;
    endcase
end

assign lpmd_xx_clk_en = clk_en || wakeup_req;
```

## 6. 中断唤醒机制

### 6.1 可唤醒中断

| 中断类型 | 描述 | 唤醒能力 |
|----------|------|----------|
| MEI | 机器外部中断 | ✓ |
| MSI | 机器软件中断 | ✓ |
| MTI | 机器定时器中断 | ✓ |
| SEI | 监督外部中断 | ✓ |
| SSI | 监督软件中断 | ✓ |
| STI | 监督定时器中断 | ✓ |

### 6.2 唤醒流程

```
1. 检测到有效中断
2. 生成唤醒请求
3. 恢复时钟
4. 等待时钟稳定
5. 开始中断处理
```

## 7. 功耗控制策略

### 7.1 进入低功耗模式

```
1. 软件设置功耗模式寄存器
2. 执行WFI指令
3. 硬件检测空闲状态
4. 进入指定低功耗模式
5. 关闭不必要的时钟
```

### 7.2 退出低功耗模式

```
1. 检测到中断
2. 生成唤醒请求
3. 恢复时钟
4. 处理中断
5. 返回正常模式
```

## 9. 设计要点

### 9.1 唤醒延迟
- 不同模式唤醒延迟不同
- Sleep模式唤醒延迟最长
- 需要考虑时钟稳定时间

### 9.2 中断保持
- 唤醒时需要保持中断信息
- 确保中断不被丢失

### 9.3 状态保存
- 进入低功耗模式前保存必要状态
- 唤醒后恢复状态
