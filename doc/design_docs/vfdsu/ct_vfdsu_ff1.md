# ct_vfdsu_ff1 模块详细设计方案

## 1. 模块概述

### 1.1 模块名称
`ct_vfdsu_ff1` - 查找第一个1（Find First One）模块

### 1.2 功能描述
`ct_vfdsu_ff1` 是VFDSU的工具模块，用于在输入数据中查找第一个（最高位）为1的位置。该模块主要用于非规格化浮点数的规格化处理，输出移位量和二进制值。

### 1.3 主要特性
- 52位输入（双精度尾数宽度）
- 输出移位量（6位）
- 输出二进制值
- 纯组合逻辑实现
- 单周期完成查找

## 2. 功能原理

### 2.1 FF1算法
FF1（Find First One）算法用于查找二进制数中第一个为1的位的位置。在浮点运算中，主要用于：

1. **非规格化数规格化**: 非规格化浮点数的尾数没有隐含的1，需要找到第一个1的位置来确定移位量
2. **前导零计数**: 确定需要左移的位数

### 2.2 规格化应用

```
非规格化数: 0.0000000010101...
                ↑ 第一个1

规格化后:   1.0101... × 2^(-8)
            ↑ 移位量 = 8
```

## 4. 接口定义

### 4.1 输入信号

| 信号名 | 位宽 | 描述 |
|--------|------|------|
| `frac[51:0]` | 52 | 输入尾数（待查找数据） |

### 4.2 输出信号

| 信号名 | 位宽 | 描述 |
|--------|------|------|
| `fanc_shift_num[5:0]` | 6 | 移位量（第一个1的位置） |
| `frac_bin_val` | 1 | 二进制值标志（表示是否找到1） |

## 5. 关键逻辑设计

### 5.1 casez实现

```verilog
always @(*) begin
    casez (frac)
        52'b1????????????????????????????????????????????????????: begin
            fanc_shift_num = 6'd0;
            frac_bin_val = 1'b1;
        end
        52'b01???????????????????????????????????????????????????: begin
            fanc_shift_num = 6'd1;
            frac_bin_val = 1'b1;
        end
        52'b001??????????????????????????????????????????????????: begin
            fanc_shift_num = 6'd2;
            frac_bin_val = 1'b1;
        end
        52'b0001?????????????????????????????????????????????????: begin
            fanc_shift_num = 6'd3;
            frac_bin_val = 1'b1;
        end
        // ... 中间情况省略
        52'b0000000000000000000000000000000000000000000000000001: begin
            fanc_shift_num = 6'd51;
            frac_bin_val = 1'b1;
        end
        default: begin
            fanc_shift_num = 6'd0;
            frac_bin_val = 1'b0;
        end
    endcase
end
```

### 5.2 移位量与位置对应

| frac中第一个1的位置 | fanc_shift_num | 含义 |
|---------------------|----------------|------|
| frac[51] | 0 | 无需移位 |
| frac[50] | 1 | 左移1位 |
| frac[49] | 2 | 左移2位 |
| ... | ... | ... |
| frac[0] | 51 | 左移51位 |
| 无1 | 0 | 全零 |

### 5.3 二进制值含义

```verilog
// frac_bin_val 表示找到的1对应的二进制值
// 用于指数调整计算

// 例如:
// frac[51] = 1 → bin_val = 1.0 (2^0)
// frac[50] = 1 → bin_val = 0.5 (2^-1)
// frac[49] = 1 → bin_val = 0.25 (2^-2)
```

## 6. 应用场景

### 6.1 非规格化数规格化

```verilog
// 在 ct_vfdsu_prepare 中使用
// 输入: 非规格化数的尾数

ct_vfdsu_ff1 u_ff1 (
    .frac           (src_frac),
    .fanc_shift_num (shift_num),
    .frac_bin_val   (bin_val)
);

// 规格化处理
assign normalized_frac = src_frac << shift_num;
assign adjusted_expt = src_expt - shift_num + 1;
```

### 6.2 指数调整

```
非规格化数: 
  expt = 0
  frac = 0.00001xxx...

FF1结果:
  shift_num = 4
  bin_val = 1

规格化后:
  expt' = 0 - 4 + 1 + BIAS = BIAS - 3
  frac' = 1.xxx...
```

## 7. 时序特性

### 7.1 组合逻辑延迟

```
输入 frac[51:0]
    │
    ▼
┌─────────────────┐
│   casez 匹配    │  组合逻辑延迟
│   优先级编码    │
└─────────────────┘
    │
    ▼
输出 fanc_shift_num[5:0], frac_bin_val
```

### 7.2 关键路径
- casez语句综合为优先级编码器
- 延迟取决于综合工具优化

## 8. 设计要点

### 8.1 优先级编码
- 使用casez实现优先级编码
- 从高位到低位依次匹配

### 8.2 边界情况
- 全零输入：shift_num=0, bin_val=0
- 最高位为1：shift_num=0, bin_val=1

### 8.3 位宽考虑
- 输入52位对应双精度尾数
- 移位量6位可表示0~63

## 9. 优化方向

### 9.1 面积优化
- 可使用二分查找结构减少逻辑
- 牺牲延迟换取面积

### 9.2 速度优化
- 使用并行前缀结构
- 减少关键路径延迟
