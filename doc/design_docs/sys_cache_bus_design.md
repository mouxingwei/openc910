# OpenC910 L2C, CIU & BIU (缓存与总线互联) 详细设计说明书

## 1. 引言与概述 (Overview)
### 1.1 模块功能定位
高速缓存一致性互联 (CIU, Coherent Interconnect Unit) 和二级缓存 (L2C) 是 OpenC910 多核（Multi-Core）簇级别的关键枢纽。它们位于各个 CPU 核心 (L1 缓存) 之外，负责将最多 4 个 C910 核心的访存请求聚合、过滤，维持它们相互之间的一致性，并通过总线接口单元 (BIU) 与外部 DDR、外设进行通信。

---

## 2. 核心二级子模块及数据流向 (Top-Level Architecture)

这三个部分（L2C, CIU, BIU）通常被封装在一个多核顶层结构下 (`gen_rtl/l2c`, `gen_rtl/ciu`, `gen_rtl/biu`)，形成芯片内的访存骨干网。

### 2.1 高级一致性互联单元 (CIU)
对应 `gen_rtl/ciu/rtl` 模块群。
- **协议管理**：硬件支持典型的 MOESI 协议。它负责侦听 (Snoop) 所有连接在其上的 C910 核心抛出的 L1 Miss 请求和写入操作。
- **侦听目录 (Snoop Filter/Directory)**：当某个核心访问一条数据时，CIU 仲裁中心会询问其他核心是否拥有此数据的 Dirty 拷贝，以决定是直接从主存拉取，还是让拥有者核心进行 Cache Intervention 写回。
- **数据通道**：提供宽带交叉开关 (Crossbar) 使得多核间的数据共享极速完成。

### 2.2 二级缓存 (L2C)
对应 `gen_rtl/l2c/rtl` 模块群。
- **功能**：由所有核心共享的大容量后备缓存。显著降低了 L1 Miss 后访问慢速外部 DDR 的惩罚。
- **架构特点**：
  - **多 Bank 切分**：为了提供足够的读写吞吐，L2 数据阵列往往切分为多个可并行独立访问的 Bank。
  - **包容性 (Inclusiveness)**：可以是部分包容或者非包容结构，依赖于 Tag 的设计，以减少 L1 和 L2 的乒乓效应。
  - **ECC 保护**：作为大容量 SRAM，配备了数据校正机制。

### 2.3 总线接口单元 (BIU)
对应 `gen_rtl/biu/rtl`。
- **功能**：作为 C910 CPU 簇与外部 SoC 总线网桥 (如 AXI4 / ACE) 的翻译官。
- **握手机制**：将 CIU 合并后的内部事务（ReadShared、MakeUnique 等）翻译为外部标准的 AXI 读写事务。处理总线上回传的乱序响应，并对对齐/分块传输进行拼装。

---

## 3. 对外接口规格 (Module Interfaces & IO)

### 3.1 对内接口 (与 Core 的 LSU/IFU)
- C910 核心的 IFU (指令流) 和 LSU (数据流) 分别拥有专用的主接口汇入 CIU。所有的缓存未命中请求在此被串行化或重排序。
- 此外，CIU 会向各个核心的 LSU (特别是 `ct_lsu_snoop_*`) 发送 Snoop Req 监听请求。

### 3.2 对外接口 (与 SoC 外设)
- 输出标准的 AXI 主设备 (Master) 接口：分为读通道 (AR, R) 与写通道 (AW, W, B)。如果 SoC 支持硬件一致性，此处则升级为 ACE 接口，含有侦听与响应附带通道。

---

## 4. 微架构亮点总结
- **高并发与防死锁设计**：在处理 L1 驱逐 (Eviction)、外部监听 (Snoop) 与新请求读写相互交织时，设计有独立的缓冲队列且在信用值 (Credit) 机制约束下运行，彻底消除协议死锁的可能。
- **流式预取合并**：CIU 可以监测出多核对连续地址的交错访问，智能地向下游主存合并为长 Burst 事务，极大提升了 DRAM 的带宽利用率。
