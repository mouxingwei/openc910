# OpenC910 IU (Integer Unit) 详细设计说明书

## 1. 引言与概述 (Overview)
### 1.1 模块功能定位
IU (Integer Unit) 是 CPU 的核心执行机构之一。它负责所有基础整数运算、分支跳转决策验证以及特殊/系统特权指令的操作。
在乱序超标量架构中，IU 从 IDU 的发射队列 (Issue Queue，如 AIQ0, AIQ1, BIQ) 接收已经解除读后写(RAW)依赖的就绪指令，并在计算完成后通过旁路网络提早交付数据或写回物理寄存器。

---

## 2. 核心二级子模块及数据流向 (Top-Level Architecture)

OpenC910 的 IU 通常被划分为多个并行的执行流水线 (Pipes)。

### 2.1 算术逻辑单元 (ALU)
对应 `ct_iu_alu`.
- 接收 `aiq0` 或 `aiq1` 派发的指令。
- **功能**：处理所有基础算数 (ADD, SUB) 与逻辑运算 (AND, OR, XOR)，移位操作 (Shift) 以及符号扩展。
- **特点**：延迟通常为单周期。在 EX 阶段得出结果后，立即广播给 IDU 的前递逻辑。

### 2.2 整数乘法器 (MULT)
对应 `ct_iu_mult`。
- **功能**：执行 32/64位有符号和无符号乘法。
- **结构**：通常是多周期的流水化乘法器 (例如 `multiplier_65x65_3_stage.v`)。
- **旁路**：乘法结果在其流水线的最后一级送回旁路网络。

### 2.3 整数除法器 (DIV)
对应 `ct_iu_div` 及衍生。
- **功能**：执行长周期的整数除法与求余指令。
- **结构**：主要基于基数 16 的 SRT 算法 (`ct_iu_div_srt_radix16`) 实现迭代求解。由于多周期不可彻底流水化（独占），故需要阻塞接口。

### 2.4 分支跳转单元 (BJU)
对应 `ct_iu_bju` 系列。
- 接收来自 `biq` 分发的条件与无条件跳转指令。
- **功能**：计算分支靶地址且评估跳转条件是否满足。
- **分支预测修正**：
  - 内部包含 PC FIFO (`ct_iu_bju_pcfifo`) 记录当初 IFU 取该指令时的各种预测信息 (如 BTB 目标、BHT 方向、RAS 栈顶等)。
  - 当 BJU 判断发生误预测 (Misprediction) 时，会发出恢复信号 (Recovery)，立刻冲刷后续流水线，并将修正的 PC 与 BHT 训练数据一并送回前端更新预测表。

### 2.5 特殊指令单元 (SPECIAL)
对应 `ct_iu_special`。
- **功能**：处理控制状态寄存器 (CSR) 的读写，同步指令 (Fence.i 等)，环境断点 (Ebreak) 以及特权级环境调用 (Ecall/mret)。

---

## 3. 对外接口规格 (Module Interfaces & IO)

### 3.1 IU 与 IDU (解码与寄存器堆)
- **输入**：`idu_iu_rf_pipe*_src0` (`_src1`, `_src2`), 及指令控制字 `func`, `iid` (指令序号)。
- **前递网络**：将执行结果通过 `iu_idu_ex*_pipe*_fwd_preg_data` 直接输送回 IDU 进行旁路读取，极大缩短数据旁路延迟。

### 3.2 IU 与 CP0 (系统协处理器)
- **CSR 通信**：`iu_cp0_epc`, `iu_cp0_vec`, 读/写使能。
- **状态查询**：获取当前的特权级别等信息以决定是否抛出非法指令异常。

### 3.3 IU 与 RTU (退休单元)
- **完成握手**：将指令执行结束的状态 (正常或发生异常) 与数据附着上该指令的 ID (`iid`) 发送至 RTU，由 RTU 裁决是否提交退休。

---

## 4. 关键设计亮点
- **快慢速执行管道分离**：将能在 1~3 周期返回的 ALU/分支/乘法与长延迟（无法完美流水化）的除法单元分开放置，防止慢速指令堵塞整个发射口，使得就绪的短指令能继续旁路乱序执行。
- **激进的分支恢复机制**：BJU 检测到误预测时，不用等到指令退休，立刻进行数据通路冲刷 (Pipeline Flush)，降低了分支预测错误带来的巨大 IPC 惩罚。
