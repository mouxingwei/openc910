# ct_rtu_encode_96 模块设计文档

## 1. 模块概述

### 1.1 功能描述
`ct_rtu_encode_96` 模块是一个独热码到二进制编码器，用于将96位独热码（One-Hot）编码转换为7位二进制数。该模块是RTU（Rename Table Unit，重命名表单元）子系统的重要组成部分，主要用于处理较大规模重命名表中的索引编码操作。

### 1.2 主要特性
- 输入：96位独热码信号
- 输出：7位二进制编码
- 纯组合逻辑实现，无时钟依赖
- 低延迟设计，单级逻辑完成编码
- 支持全范围编码（0-95）

### 1.3 应用场景
- 大规模重命名表索引编码
- 扩展物理寄存器索引生成
- 多端口仲裁器优先级编码
- 复杂多路选择器控制信号生成

## 2. 接口说明

### 2.1 端口列表

| 端口名称 | 方向 | 位宽 | 类型 | 描述 |
|---------|------|------|------|------|
| `x_num_expand` | Input | 96 | wire | 96位独热码输入信号 |
| `x_num` | Output | 7 | wire | 7位二进制编码输出 |

### 2.2 端口详细说明

#### 2.2.1 输入端口

**x_num_expand[95:0]**
- 类型：96位独热码
- 功能：表示需要编码的位置索引
- 编码规则：同一时刻仅允许1位为高电平
- 示例：
  - `96'h0000_0000_0000_0000_0000_0001` → 表示索引0
  - `96'h0000_0000_0000_0000_0000_0002` → 表示索引1
  - `96'h8000_0000_0000_0000_0000_0000` → 表示索引95

#### 2.2.2 输出端口

**x_num[6:0]**
- 类型：7位二进制数
- 功能：输出独热码对应的二进制索引值
- 取值范围：0-95
- 示例：
  - 输入 `x_num_expand[0]=1` → 输出 `x_num=7'd0`
  - 输入 `x_num_expand[1]=1` → 输出 `x_num=7'd1`
  - 输入 `x_num_expand[95]=1` → 输出 `x_num=7'd95`

## 4. 关键逻辑说明

### 4.1 编码原理

该模块采用**并行比较+或运算**的方式实现独热码到二进制的编码：

```
x_num[6:0] = {7{x_num_expand[0]}}  & 7'd0  |
             {7{x_num_expand[1]}}  & 7'd1  |
             {7{x_num_expand[2]}}  & 7'd2  |
             ...                            |
             {7{x_num_expand[95]}} & 7'd95
```

### 4.2 编码逻辑详解

#### 4.2.1 位扩展操作
- `{7{x_num_expand[i]}}`：将单比特信号复制7次
- 例如：`x_num_expand[5]=1` → `{7{1'b1}} = 7'b1111111`

#### 4.2.2 条件选择
- `& 7'dN`：与对应的二进制值进行与运算
- 例如：`x_num_expand[5]=1` → `7'b1111111 & 7'd5 = 7'd5`

#### 4.2.3 结果合并
- 所有96路结果通过或运算合并
- 由于输入是独热码，同一时刻只有一路有效

### 4.3 时序特性

| 参数 | 描述 | 典型值 |
|------|------|--------|
| Tpd | 传播延迟 | 取决于综合优化 |
| 逻辑深度 | 组合逻辑级数 | 1级（并行结构） |
| 关键路径 | 96输入或树 | 工具自动优化 |

### 4.4 设计优势

1. **并行处理**：所有96个输入同时检测，无优先级冲突
2. **低延迟**：单级逻辑实现，无流水线开销
3. **扩展性强**：支持96个条目的编码需求
4. **可综合性**：纯组合逻辑，易于综合和时序收敛

## 7. 设计注意事项

### 7.1 输入约束
- 输入必须为有效的独热码（同一时刻仅1位为高）
- 若输入全0或多位为高，输出结果不可预测

### 7.2 时序考虑
- 该模块为组合逻辑，输出可能产生毛刺
- 建议在输出端添加寄存器采样
- 在高速设计中需关注关键路径延迟

### 7.3 综合优化
- 综合工具会自动优化编码逻辑
- 可能优化为优先编码器或树形编码器
- 面积和时序可通过约束文件控制

### 7.4 与encode_64的区别
- 输入位宽更大（96位 vs 64位）
- 输出位宽更大（7位 vs 6位）
- 适用于更大规模的寄存器重命名场景
