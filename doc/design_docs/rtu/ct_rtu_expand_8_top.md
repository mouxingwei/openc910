# ct_rtu_expand_8 模块设计文档

## 1. 模块概述

### 1.1 功能描述
`ct_rtu_expand_8` 模块是一个二进制到独热码扩展器，用于将3位二进制数扩展为8位独热码（One-Hot）。该模块是RTU（Rename Table Unit，重命名表单元）子系统的重要组成部分，主要用于处理重命名表中的索引解码操作。

### 1.2 主要特性
- 输入：3位二进制数
- 输出：8位独热码
- 纯组合逻辑实现，无时钟依赖
- 低延迟设计，单级逻辑完成扩展
- 支持全范围解码（0-7）

### 1.3 应用场景
- 重命名表索引解码
- 物理寄存器选择信号生成
- 多路选择器使能信号生成
- 译码器输出信号扩展

## 2. 接口说明

### 2.1 端口列表

| 端口名称 | 方向 | 位宽 | 类型 | 描述 |
|---------|------|------|------|------|
| `x_num` | Input | 3 | wire | 3位二进制输入 |
| `x_num_expand` | Output | 8 | wire | 8位独热码输出 |

### 2.2 端口详细说明

#### 2.2.1 输入端口

**x_num[2:0]**
- 类型：3位二进制数
- 功能：表示需要扩展的索引值
- 取值范围：0-7
- 示例：
  - `3'd0` → 表示索引0
  - `3'd1` → 表示索引1
  - `3'd7` → 表示索引7

#### 2.2.2 输出端口

**x_num_expand[7:0]**
- 类型：8位独热码
- 功能：输出二进制数对应的独热码
- 编码规则：输出仅有1位为高电平
- 示例：
  - 输入 `x_num=3'd0` → 输出 `8'b0000_0001`
  - 输入 `x_num=3'd1` → 输出 `8'b0000_0010`
  - 输入 `x_num=3'd7` → 输出 `8'b1000_0000`

## 4. 关键逻辑说明

### 4.1 扩展原理

该模块采用**相等比较**的方式实现二进制到独热码的扩展：

```verilog
assign x_num_expand[0] = (x_num[2:0] == 3'd0);
assign x_num_expand[1] = (x_num[2:0] == 3'd1);
assign x_num_expand[2] = (x_num[2:0] == 3'd2);
assign x_num_expand[3] = (x_num[2:0] == 3'd3);
assign x_num_expand[4] = (x_num[2:0] == 3'd4);
assign x_num_expand[5] = (x_num[2:0] == 3'd5);
assign x_num_expand[6] = (x_num[2:0] == 3'd6);
assign x_num_expand[7] = (x_num[2:0] == 3'd7);
```

### 4.2 扩展逻辑详解

#### 4.2.1 比较操作
- 每个输出位对应一个相等比较器
- 当输入值等于对应的索引时，该位输出为1
- 例如：`x_num=3'd5` → `x_num_expand[5]=1`，其他位为0

#### 4.2.2 独热码特性
- 同一时刻仅1位输出为高
- 输出位宽 = 2^输入位宽
- 解码延迟固定，无优先级依赖

### 4.3 时序特性

| 参数 | 描述 | 典型值 |
|------|------|--------|
| Tpd | 传播延迟 | 取决于综合优化 |
| 逻辑深度 | 组合逻辑级数 | 1级（并行比较） |
| 关键路径 | 3位比较器 | 工具自动优化 |

### 4.4 设计优势

1. **并行解码**：所有比较同时进行，无优先级
2. **低延迟**：单级比较器实现
3. **面积小**：仅8个简单比较器
4. **可综合性**：纯组合逻辑，易于综合

## 7. 设计注意事项

### 7.1 输入约束
- 输入应为有效的3位二进制数（0-7）
- 无需特殊约束，所有输入值都有对应输出

### 7.2 时序考虑
- 该模块为组合逻辑，输出稳定无毛刺
- 可直接连接到后续逻辑
- 在高速设计中需关注比较器延迟

### 7.3 综合优化
- 综合工具会优化为译码器结构
- 可能使用标准单元库中的译码器单元
- 面积和时序可通过约束文件控制
