# ct_biu_write_channel 模块详细设计方案

## 1. 模块概述

### 1.1 模块名称
`ct_biu_write_channel` - AXI写通道模块

### 1.2 功能描述
`ct_biu_write_channel` 是BIU的AXI写通道模块，负责处理处理器的写请求。该模块实现AXI4协议的AW（写地址）通道、W（写数据）通道和B（写响应）通道，管理写请求的发送、写数据的传输和写响应的接收。

### 1.3 主要特性
- AXI4 AW通道实现
- AXI4 W通道实现
- AXI4 B通道实现
- 突发传输支持
- Victim和Store分离处理
- 时钟门控支持

## 2. AXI写通道协议

### 2.1 AW通道信号

| 信号 | 方向 | 描述 |
|------|------|------|
| AWVALID | 输出 | 写地址有效 |
| AWREADY | 输入 | 写地址就绪 |
| AWADDR | 输出 | 写地址 |
| AWID | 输出 | 写ID |
| AWSIZE | 输出 | 传输大小 |
| AWLEN | 输出 | 突发长度 |
| AWBURST | 输出 | 突发类型 |
| AWLOCK | 输出 | 锁类型 |
| AWCACHE | 输出 | 缓存属性 |
| AWPROT | 输出 | 保护属性 |

### 2.2 W通道信号

| 信号 | 方向 | 描述 |
|------|------|------|
| WVALID | 输出 | 写数据有效 |
| WREADY | 输入 | 写数据就绪 |
| WDATA | 输出 | 写数据 |
| WSTRB | 输出 | 写数据选通 |
| WLAST | 输出 | 最后数据 |

### 2.3 B通道信号

| 信号 | 方向 | 描述 |
|------|------|------|
| BVALID | 输入 | 写响应有效 |
| BREADY | 输出 | 写响应就绪 |
| BRESP | 输入 | 写响应 |
| BID | 输入 | 写ID |

## 6. 关键逻辑设计

### 6.1 AW通道握手

```verilog
// AWVALID生成
assign biu_pad_awvalid = aw_state == AW_REQ;

// 地址输出
assign biu_pad_awaddr = aw_addr_buf;
assign biu_pad_awid = aw_id_buf;

// 握手成功
assign aw_handshake = biu_pad_awvalid && pad_biu_awready;
```

### 6.2 W通道握手

```verilog
// WVALID生成
assign biu_pad_wvalid = w_valid_buf;

// 数据输出
assign biu_pad_wdata = w_data_buf;
assign biu_pad_wstrb = w_strb_buf;
assign biu_pad_wlast = w_last_buf;

// 握手成功
assign w_handshake = biu_pad_wvalid && pad_biu_wready;
```

### 6.3 B通道握手

```verilog
// BREADY生成
assign biu_pad_bready = 1'b1;

// 响应接收
always @(posedge clk) begin
    if (pad_biu_bvalid && biu_pad_bready) begin
        b_resp_buf <= pad_biu_bresp;
        b_id_buf <= pad_biu_bid;
    end
end
```

### 6.4 时钟使能

```verilog
assign vict_aw_clk_en = vict_aw_req || (vict_aw_state != IDLE);
assign st_aw_clk_en = st_aw_req || (st_aw_state != IDLE);
assign vict_w_clk_en = vict_w_valid || w_handshake;
assign st_w_clk_en = st_w_valid || w_handshake;
assign write_b_clk_en = pad_biu_bvalid || b_valid_buf;
```

## 8. 设计要点

### 8.1 Victim vs Store
- Victim: Cache驱逐写操作
- Store: 普通存储操作
- 分离处理以提高效率

### 8.2 突发传输
- 支持AXI突发传输
- 正确生成WLAST信号

### 8.3 响应处理
- 检测BRESP错误
- 传递错误信息
