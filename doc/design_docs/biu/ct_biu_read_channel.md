# ct_biu_read_channel 模块详细设计方案

## 1. 模块概述

### 1.1 模块名称
`ct_biu_read_channel` - AXI读通道模块

### 1.2 功能描述
`ct_biu_read_channel` 是BIU的AXI读通道模块，负责处理处理器的读请求。该模块实现AXI4协议的AR（读地址）通道和R（读数据）通道，管理读请求的发送和读数据的接收。

### 1.3 主要特性
- AXI4 AR通道实现
- AXI4 R通道实现
- 突发传输支持
- 数据缓冲
- 时钟门控支持

## 2. AXI读通道协议

### 2.1 AR通道信号

| 信号 | 方向 | 描述 |
|------|------|------|
| ARVALID | 输出 | 读地址有效 |
| ARREADY | 输入 | 读地址就绪 |
| ARADDR | 输出 | 读地址 |
| ARID | 输出 | 读ID |
| ARSIZE | 输出 | 传输大小 |
| ARLEN | 输出 | 突发长度 |
| ARBURST | 输出 | 突发类型 |
| ARLOCK | 输出 | 锁类型 |
| ARCACHE | 输出 | 缓存属性 |
| ARPROT | 输出 | 保护属性 |

### 2.2 R通道信号

| 信号 | 方向 | 描述 |
|------|------|------|
| RVALID | 输入 | 读数据有效 |
| RREADY | 输出 | 读数据就绪 |
| RDATA | 输入 | 读数据 |
| RLAST | 输入 | 最后数据 |
| RID | 输入 | 读ID |
| RRESP | 输入 | 读响应 |

## 5. 状态机设计

### 5.1 AR通道状态机

```
          ┌─────────────────────────────────────┐
          │                                     │
          ▼              arready               │
    ┌───────────┐      (地址被接受)             │
    │   IDLE    │◄────────────────────────────┐ │
    │           │                             │ │
    └───────────┘                             │ │
          │                                   │ │
          │ ar_req                            │ │
          │ (有读请求)                         │ │
          ▼                                   │ │
    ┌───────────┐                             │ │
    │    REQ    │                             │ │
    │           │─────────────────────────────┘ │
    └───────────┘                               │
          │                                     │
          │ arvalid && arready                  │
          │ (握手成功)                           │
          ▼                                     │
    ┌───────────┐                               │
    │   WAIT    │                               │
    │ (等待数据) │                               │
    └───────────┘                               │
          │                                     │
          │ rvalid && rlast                     │
          │ (数据接收完成)                       │
          └─────────────────────────────────────┘
```

### 5.2 状态定义

| 状态 | 编码 | 描述 |
|------|------|------|
| IDLE | 2'b00 | 空闲状态 |
| REQ | 2'b01 | 发送请求 |
| WAIT | 2'b10 | 等待数据 |
| DONE | 2'b11 | 完成 |

## 6. 关键逻辑设计

### 6.1 AR通道握手

```verilog
// ARVALID生成
assign biu_pad_arvalid = ar_state == REQ;

// 地址输出
assign biu_pad_araddr = ar_addr_buf;
assign biu_pad_arid = ar_id_buf;

// 握手成功
assign ar_handshake = biu_pad_arvalid && pad_biu_arready;
```

### 6.2 R通道握手

```verilog
// RREADY生成
assign biu_pad_rready = 1'b1;  // 总是准备好接收

// 数据接收
always @(posedge clk) begin
    if (pad_biu_rvalid && biu_pad_rready) begin
        r_data_buf <= pad_biu_rdata;
        r_last_buf <= pad_biu_rlast;
        r_id_buf <= pad_biu_rid;
    end
end

// 数据有效输出
assign biu_lsu_rvalid = pad_biu_rvalid;
assign biu_lsu_rdata = pad_biu_rdata;
assign biu_lsu_rlast = pad_biu_rlast;
```

### 6.3 时钟使能

```verilog
assign read_ar_clk_en = lsu_biu_ar_req || (ar_state != IDLE);
assign read_r_clk_en = pad_biu_rvalid || (r_data_valid);
```
