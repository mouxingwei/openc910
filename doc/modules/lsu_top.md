# LSU (Load Store Unit) 访存单元

## 1. 模块功能说明

LSU (Load Store Unit) 是 OpenC910 处理器的访存单元，负责执行所有内存访问操作，包括 Load、Store、原子操作和缓存管理指令。

### 主要功能

- **Load 操作**: 执行内存读取操作，支持多种数据宽度
- **Store 操作**: 执行内存写入操作，支持 Store Buffer
- **原子操作**: 支持 AMO (Atomic Memory Operations) 指令
- **缓存管理**: 支持 Cache 操作指令 (Flush, Invalidate, Clean)
- **地址翻译**: 与 MMU 协作完成虚拟地址翻译
- **内存一致性**: 维护内存一致性，支持 Fence 指令

### 主要特性

| 特性 | 描述 |
|------|------|
| DCache 容量 | 32KB, 4路组相联 |
| DCache 行大小 | 64 Bytes |
| Load Queue 深度 | 16 条目 |
| Store Queue 深度 | 16 条目 |
| Store Buffer 深度 | 8 条目 |
| Line Fill Buffer | 4 条目 |
| 写回缓冲 | 8 条目 |
| 预取支持 | 支持 L1 和 L2 预取 |

## 4. 模块实现方案

### 4.1 访存流水线

LSU 采用多级流水线结构：

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│   AG    │──►│   DA    │──►│   DC    │──►│   WB    │
│地址生成 │   │数据访问 │   │数据校验 │   │写回    │
└─────────┘   └─────────┘   └─────────┘   └─────────┘
```

**各级功能**:
- **AG (Address Generation)**: 计算内存地址，发起 TLB 查询
- **DA (Data Access)**: 访问 DCache，处理 Cache 命中/缺失
- **DC (Data Check)**: 数据校验，处理异常
- **WB (Write Back)**: 写回结果，前递数据
