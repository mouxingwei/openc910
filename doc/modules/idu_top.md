# IDU (Instruction Decode Unit) 译码单元

## 1. 模块功能说明

IDU (Instruction Decode Unit) 是 OpenC910 处理器的译码单元，负责对 IFU 获取的指令进行译码、重命名和发射。IDU 实现了高性能的指令调度机制，支持乱序执行。

### 主要功能

- **指令译码**: 对 RISC-V 指令进行完整译码，生成控制信号
- **寄存器重命名**: 将逻辑寄存器映射到物理寄存器，消除 WAW 和 WAR 依赖
- **指令发射**: 将就绪指令发射到各执行单元
- **依赖检测**: 检测指令间的数据依赖关系
- **指令队列管理**: 管理各类指令队列 (AIQ, BIQ, LSIQ, VIQ 等)

### 主要特性

| 特性 | 描述 |
|------|------|
| 发射宽度 | 每周期最多发射 4 条指令 |
| 物理寄存器数 | 128 个通用物理寄存器 |
| 向量寄存器 | 32 个向量寄存器 (支持 RVV) |
| ROB 条目数 | 128 条目 |
| AIQ 条目数 | 16 条目 × 2 |
| BIQ 条目数 | 12 条目 |
| LSIQ 条目数 | 12 条目 |
| VIQ 条目数 | 16 条目 × 2 |

## 4. 模块实现方案

### 4.1 译码流水线

IDU 采用多级流水线结构：

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│   ID    │──►│   IR    │──►│   IS    │──►│   RF    │
│  译码   │   │  重命名  │   │  发射   │   │ 读寄存器│
└─────────┘   └─────────┘   └─────────┘   └─────────┘
```

**各级功能**:
- **ID (Instruction Decode)**: 指令译码，生成控制信号
- **IR (Instruction Rename)**: 寄存器重命名，分配物理寄存器
- **IS (Instruction Schedule)**: 指令调度，发射到执行单元
- **RF (Register Fetch)**: 读取寄存器值，前递处理

### 4.2 寄存器重命名机制

```
┌─────────────────────────────────────────────────────────────────────┐
│                      寄存器重命名结构                                 │
│                                                                     │
│  ┌─────────────────┐     ┌─────────────────┐     ┌───────────────┐ │
│  │  Architectural  │     │    Physical     │     │   Free List   │ │
│  │  Register Map   │────►│   Register File │────►│   (空闲列表)  │ │
│  │  (逻辑→物理)    │     │   (128 entries) │     │               │ │
│  └─────────────────┘     └─────────────────┘     └───────────────┘ │
│          │                       │                               │
│          ▼                       ▼                               │
│  ┌─────────────────┐     ┌─────────────────┐                     │
│  │   Dependency    │     │    Forwarding   │                     │
│  │   Check Table   │     │     Network     │                     │
│  │   (依赖检测)    │     │    (前递网络)   │                     │
│  └─────────────────┘     └─────────────────┘                     │
└─────────────────────────────────────────────────────────────────────┘
```

**重命名过程**:
1. 读取源操作数的物理寄存器映射
2. 检测源操作数是否就绪 (依赖检测)
3. 分配新的目标物理寄存器
4. 更新映射表
5. 创建 ROB 条目
