## 1. 架构基础与 CSR 寄存器变更

### 1.1 状态管理与 vtype 编码

#### 1.1.1 vtype 寄存器重构

**v1.0 引入 Agnostic（不可知）策略字段：**

| 字段 | 位宽 | v0.7.1 | v1.0 | 说明 |
|------|------|--------|------|------|
| vma | 1 | 不存在 | 新增 | Mask Agnostic，掩码忽略元素的处理策略 |
| vta | 1 | 不存在 | 新增 | Tail Agnostic，尾部元素的处理策略 |
| vediv | 2 | 存在 | **移除** | v1.0 不再支持元素除法 |

**Agnostic 策略详解：**

```
vta/vma = 0: 尾部/掩码忽略元素必须保持原值（确定性）
vta/vma = 1: 尾部/掩码忽略元素可以是原值或全1（非确定性）
```

**硬件实现影响：**
- **优势**：允许寄存器重命名时无需初始化尾部元素，降低功耗
- **劣势**：软件调试时无法依赖尾部元素的确定值
- **验证要点**：需测试两种策略下的功能正确性

#### 1.1.2 LMUL 分数化支持

**v1.0 LMUL 编码表（vlmul[2:0]）：**

| 编码 | LMUL 值 | 说明 |
|------|---------|------|
| 000 | 1 | 标准模式 |
| 001 | 2 | 扩展模式 |
| 010 | 4 | 扩展模式 |
| 011 | 8 | 扩展模式 |
| 101 | 1/2 | **v1.0 新增** |
| 110 | 1/4 | **v1.0 新增** |
| 111 | 1/8 | **v1.0 新增** |

**Fractional LMUL 应用场景：**

```c
// v0.7.1: 处理 SEW=64 数据时，LMUL 最小为 1，占用整个寄存器组
vsetvli t0, a0, e64, m1  // 占用 1 个向量寄存器

// v1.0: 使用 LMUL=1/2，仅占用半个寄存器
vsetvli t0, a0, e64, mf2  // 占用 0.5 个向量寄存器，节省资源
```

**优势：**
- 混合位宽运算时减少寄存器压力
- 支持更灵活的向量分组策略

**硬件实现要点：**
- 需要支持部分寄存器写使能
- 读写逻辑需处理非对齐的寄存器边界

#### 1.1.3 新增 vlenb CSR

**v1.0 新增只读寄存器：**

| 寄存器 | 地址 | 位宽 | 说明 |
|--------|------|------|------|
| vlenb | 0xC22 | XLEN | 向量长度（以字节为单位） |

**使用对比：**

```c
// v0.7.1: 需要通过 vsetvli 间接获取
vsetvli t0, zero, e8, m1  // 设置 SEW=8, LMUL=1
// t0 = VL，需要计算 VLEN = VL * LMUL

// v1.0: 直接读取
csrr t0, vlenb  // t0 = VLEN / 8（字节数）
```

**应用场景：**
- 编译器优化：无需改变当前向量状态即可获取硬件参数
- OS 上下文切换：快速判断需要保存的向量寄存器数量
- 动态库加载：运行时检测硬件能力

### 1.2 vstart 寄存器语义变化

**v1.0 明确 vstart 的复位行为：**

| 场景 | v0.7.1 | v1.0 |
|------|--------|------|
| 指令正常完成 | 未明确 | vstart 必须清零 |
| 陷阱（Trap）处理 | 未明确 | vstart 保存当前元素索引 |
| 复位后 | 未明确 | vstart 必须为零 |

**硬件实现要求：**
- 每条向量指令完成时必须将 vstart 清零
- 异常发生时，vstart 指向引发异常的元素索引
- 支持 vstart > 0 的指令恢复执行

## 2. 内存访问指令的重大不兼容变更

### 2.1 索引加载/存储（Indexed Instructions）

#### 2.1.1 指令重命名与解耦

**指令对比表：**

| v0.7.1 指令 | v1.0 指令 | 关键差异 |
|-------------|-----------|----------|
| `vlxb.v` | `vluxei<eew>.v` | 索引位宽由 SEW 决定 → 由指令名决定 |
| `vlxh.v` | `vluxei16.v` | 同上 |
| `vlxw.v` | `vluxei32.v` | 同上 |
| `vlxd.v` | `vluxei64.v` | 同上 |
| `vsxb.v` | `vsuxei<eew>.v` | 同上 |

**详细示例：**

```c
// v0.7.1: 索引位宽必须等于 SEW
vsetvli t0, a0, e32, m1  // SEW=32
vlxw.v v1, (a1), v2      // 索引必须是 32 位

// v1.0: 索引位宽独立于 SEW
vsetvli t0, a0, e64, m1  // SEW=64
vluxei32.v v1, (a1), v2  // 索引可以是 32 位，数据是 64 位
```

**硬件实现影响：**
- **优势**：处理结构化数据（如结构体数组）更灵活
- **劣势**：需要额外的索引位宽解码逻辑
- **验证要点**：测试所有 EEW 组合的正确性

#### 2.1.2 有序与无序语义

**v1.0 区分有序与无序访问：**

| 指令类型 | 指令名 | 内存序语义 |
|----------|--------|-----------|
| 无序索引加载 | `vluxei<eew>.v` | 无地址依赖保证 |
| 有序索引加载 | `vloxei<eew>.v` | 保证地址访问顺序 |
| 无序索引存储 | `vsuxei<eew>.v` | 无地址依赖保证 |
| 有序索引存储 | `vsoxei<eew>.v` | 保证地址访问顺序 |

**应用场景：**
- **无序访问**：常规向量运算，性能优先
- **有序访问**：访问设备寄存器、共享内存同步

**硬件实现要点：**
- 有序访问需要禁止内存访问重排序
- 无序访问可以使用 Load/Store Queue 优化

### 2.2 全寄存器指令（Whole Register Instructions）

#### 2.2.1 新增指令列表

| 指令 | 功能 | 忽略字段 |
|------|------|----------|
| `vl1re8.v` | 加载 1 个向量寄存器（EEW=8） | vl, vtype |
| `vl1re16.v` | 加载 1 个向量寄存器（EEW=16） | vl, vtype |
| `vl1re32.v` | 加载 1 个向量寄存器（EEW=32） | vl, vtype |
| `vl1re64.v` | 加载 1 个向量寄存器（EEW=64） | vl, vtype |
| `vl2r.v` | 加载 2 个向量寄存器 | vl, vtype |
| `vl4r.v` | 加载 4 个向量寄存器 | vl, vtype |
| `vl8r.v` | 加载 8 个向量寄存器 | vl, vtype |
| `vs1r.v` | 存储 1 个向量寄存器 | vl, vtype |
| `vs2r.v` | 存储 2 个向量寄存器 | vl, vtype |
| `vs4r.v` | 存储 4 个向量寄存器 | vl, vtype |
| `vs8r.v` | 存储 8 个向量寄存器 | vl, vtype |

#### 2.2.2 关键语义

**核心特性：**
1. **忽略 vl 和 vtype**：始终操作整个寄存器
2. **固定数据长度**：等于 VLEN * nreg 字节
3. **无掩码**：不支持掩码操作

**应用场景示例：**

```c
// v0.7.1: 函数调用时保存向量寄存器
addi sp, sp, -128
vsetvli t0, zero, e8, m1  // 需要先设置 vtype
vse8.v v1, (sp)           // 保存 v1
// 问题：如果 v1 包含 SEW=64 的数据，会被截断

// v1.0: 使用全寄存器指令
addi sp, sp, -128
vs1r.v v1, (sp)           // 直接保存整个寄存器，无需设置 vtype
```

**OS 上下文切换优化：**

```c
// v1.0: 快速保存所有向量寄存器
save_vector_context:
    csrr t0, vlenb
    slli t0, t0, 3       // t0 = VLEN (bits)
    // 保存 v0-v31
    vs1r.v v0, (sp)
    addi sp, sp, t0
    vs1r.v v1, (sp)
    addi sp, sp, t0
    // ... 重复 32 次
```

**硬件实现要点：**
- 需要独立的指令解码路径
- 内存访问长度 = VLEN * nreg / 8
- 不经过向量控制逻辑（vl, vtype）

### 2.3 掩码加载/存储（Mask Load/Store）

#### 2.3.1 语义变化

**v0.7.1 vs v1.0：**

| 特性 | v0.7.1 | v1.0 |
|------|--------|------|
| 指令名 | `vlm.v` / `vsm.v` | 相同 |
| 有效元素位宽（EEW） | 随 MLEN 变化 | 固定为 8 |
| 加载长度 | ceil(VL * MLEN / 8) | ceil(VL / 8) |
| 掩码布局 | 分散布局 | 连续布局 |

**示例：**

```c
// v1.0: 加载 32 个元素的掩码
vsetvli t0, zero, e32, m1  // VL=32
vlm.v v0, (a0)             // 加载 4 字节（32 位掩码）

// v0.7.1: 相同操作可能需要更多字节
// 取决于 MLEN 的值
```

**硬件实现简化：**
- 固定 EEW=8，无需根据 SEW 调整
- 内存访问长度计算简单：ceil(VL / 8)

## 3. 掩码（Mask）与元素布局简化

### 3.1 掩码位布局（Mask Layout）

#### 3.1.1 v0.7.1 的 MLEN 逻辑

**掩码位位置计算（v0.7.1）：**

```
掩码位位置 = i * MLEN / SEW
其中：
- i: 元素索引
- MLEN: 掩码元素长度（通常等于 SEW）
- SEW: 标准元素宽度
```

**示例（v0.7.1）：**

```c
// SEW=32, LMUL=1, VLEN=128
// 元素数量 = 128 / 32 = 4
// 掩码位位置：
// 元素 0: 位 0
// 元素 1: 位 1
// 元素 2: 位 2
// 元素 3: 位 3

// SEW=16, LMUL=1, VLEN=128
// 元素数量 = 128 / 16 = 8
// 掩码位位置：
// 元素 0: 位 0
// 元素 1: 位 2  // 注意：间隔为 2
// 元素 2: 位 4
// ...
```

**问题：**
- 软件需要复杂的位偏移计算
- 不同 SEW 间掩码不兼容

#### 3.1.2 v1.0 的固定布局

**掩码位位置计算（v1.0）：**

```
掩码位位置 = i
无论 SEW 为多少，第 i 个元素的掩码位始终在第 i 位
```

**示例（v1.0）：**

```c
// 任何 SEW，掩码布局相同
// 元素 0: 位 0
// 元素 1: 位 1
// 元素 2: 位 2
// ...
```

**优势：**
- 掩码生成逻辑统一
- 不同 SEW 间掩码可复用
- 简化编译器代码生成

### 3.2 掩码处理逻辑

#### 3.2.1 Mask-Agnostic 策略

**v1.0 新增 vma 字段：**

```c
// vma = 0: 被屏蔽元素保持原值（确定性）
vadd.vv v1, v2, v3, v0.t  // v0 掩码为 0 的位置，v1 保持原值

// vma = 1: 被屏蔽元素可以是原值或全 1（非确定性）
// 硬件可以选择：
// 1. 保持原值（节省功耗）
// 2. 填充全 1（简化寄存器重命名）
```

**硬件实现优化：**

```
// 传统实现（vma=0）：
// 需要读-修改-写序列
1. 读取目标寄存器旧值
2. 根据 SEW 选择元素
3. 写入未被屏蔽的元素
4. 保持被屏蔽元素不变

// Agnostic 实现（vma=1）：
// 可以直接写入
1. 计算结果
2. 直接写入整个寄存器（被屏蔽元素可以是任意值）
```

**验证要点：**
- 测试 vma=0 时的确定性
- 测试 vma=1 时的两种可能结果
- 验证寄存器重命名逻辑

## 4. 新增功能指令（v0.7.1 不具备）

### 4.1 整数扩展指令

#### 4.1.1 指令列表

| 指令 | 功能 | 扩展比例 |
|------|------|----------|
| `vzext.vf2` | 零扩展 2 倍 | 8→16, 16→32, 32→64 |
| `vzext.vf4` | 零扩展 4 倍 | 8→32, 16→64 |
| `vzext.vf8` | 零扩展 8 倍 | 8→64 |
| `vsext.vf2` | 符号扩展 2 倍 | 8→16, 16→32, 32→64 |
| `vsext.vf4` | 符号扩展 4 倍 | 8→32, 16→64 |
| `vsext.vf8` | 符号扩展 8 倍 | 8→64 |

#### 4.1.2 使用示例

**v0.7.1 实现（繁琐）：**

```c
// 将 8 位无符号数扩展为 32 位
vsetvli t0, a0, e32, m1
// 方法 1: 使用 AND 屏蔽
vand.vx v1, v2, 0xFF
// 方法 2: 使用移位
vsll.vi v1, v2, 24
vsrl.vi v1, v1, 24
```

**v1.0 实现（简洁）：**

```c
// 将 8 位无符号数扩展为 32 位
vsetvli t0, a0, e32, m1
vzext.vf2 v1, v2  // 一条指令完成
```

**性能对比：**

| 方法 | 指令数 | 延迟 | 吞吐量 |
|------|--------|------|--------|
| v0.7.1 (AND) | 1 | 1 周期 | 1 结果/周期 |
| v0.7.1 (移位) | 2 | 2 周期 | 0.5 结果/周期 |
| v1.0 (vzext) | 1 | 1 周期 | 1 结果/周期 |

### 4.2 浮点估计指令

#### 4.2.1 指令列表

| 指令 | 功能 | 精度 |
|------|------|------|
| `vfrec7.v` | 倒数估计 | 7 位精度 |
| `vfrsqrt7.v` | 平方根倒数估计 | 7 位精度 |

#### 4.2.2 应用场景

**高性能计算（HPC）中的迭代求逆：**

```c
// 使用 Newton-Raphson 迭代求高精度倒数
// 初始值：7 位精度
vfrec7.v v1, v2

// 第一次迭代：精度提升到 ~14 位
vfmul.vv v3, v1, v2      // v3 = approx * x
vfrsub.vi v3, v3, 1      // v3 = 1 - approx * x
vfmul.vv v1, v1, v3      // v1 = approx * (1 - approx * x)

// 第二次迭代：精度提升到 ~28 位
vfmul.vv v3, v1, v2
vfrsub.vi v3, v3, 1
vfmul.vv v1, v1, v3
```

**性能优势：**
- 避免使用高延迟的除法指令（通常 20+ 周期）
- 估计指令延迟通常为 3-5 周期
- 适合大规模向量运算

### 4.3 向量掩码群体计数增强

#### 4.3.1 指令列表

| 指令 | 功能 | 输出 |
|------|------|------|
| `vcpop.m` | 计算掩码中 1 的个数 | 标量寄存器 |
| `vfirst.m` | 找到第一个 1 的位置 | 标量寄存器 |
| `vmsbf.m` | 找到第一个 1 之前的所有位 | 向量寄存器 |
| `vmsif.m` | 找到第一个 1 及之前的所有位 | 向量寄存器 |
| `vmsof.m` | 找到第一个 1 | 向量寄存器 |

#### 4.3.2 使用示例

```c
// 统计满足条件的元素数量
vmslt.vx v0, v1, x1      // v0[i] = (v1[i] < x1) ? 1 : 0
vcpop.m a0, v0           // a0 = 满足条件的元素数量

// 找到第一个满足条件的元素
vmslt.vx v0, v1, x1
vfirst.m a0, v0          // a0 = 第一个满足条件的索引
```

## 5. 扩展子集 Zve* 的细化与约束

### 5.1 Zve 扩展家族

**v1.0 正式引入嵌入式向量扩展：**

| 扩展名 | 最小 VLEN | 最大 SEW | 浮点支持 | 标量浮点要求 |
|--------|-----------|----------|----------|--------------|
| Zve32x | 32 | 32 | 无 | 无 |
| Zve32f | 32 | 32 | 单精度 | 需要 F 扩展 |
| Zve64x | 64 | 64 | 无 | 无 |
| Zve64f | 64 | 64 | 单精度 | 需要 F 扩展 |
| Zve64d | 64 | 64 | 双精度 | 需要 D 扩展 |
| V (完整) | 128 | 64 | 全部 | 需要 F/D 扩展 |

### 5.2 关键约束

#### 5.2.1 SEW 限制

**Zve32* 扩展禁止 SEW=64：**

```c
// Zve32x/f: 非法指令
vsetvli t0, a0, e64, m1  // 触发非法指令异常

// Zve64x/f/d: 合法
vsetvli t0, a0, e64, m1  // 正常执行
```

**硬件实现要点：**
- 需要在 vsetvli 执行时检查 SEW 合法性
- 非法 SEW 应触发非法指令异常

#### 5.2.2 LMUL 限制

**Zve32* 扩展的 LMUL 约束：**

| LMUL | Zve32x/f | Zve64x/f/d | V (完整) |
|------|----------|------------|----------|
| 1/8 | 支持 | 支持 | 支持 |
| 1/4 | 支持 | 支持 | 支持 |
| 1/2 | 支持 | 支持 | 支持 |
| 1 | 支持 | 支持 | 支持 |
| 2 | 支持 | 支持 | 支持 |
| 4 | 支持 | 支持 | 支持 |
| 8 | **不支持** | 支持 | 支持 |

**原因：**
- Zve32* 的 VLEN 最小为 32
- LMUL=8 时，需要 VLEN >= SEW * LMUL = 32 * 8 = 256 位
- 与最小 VLEN=32 冲突

### 5.3 与 F/D 扩展的脱钩

**Zve32x/64x 不需要标量浮点寄存器：**

```c
// Zve32x: 可以在没有 F 扩展的情况下使用向量整数运算
vadd.vv v1, v2, v3      // 合法

// Zve32x: 不能使用向量浮点指令
vfadd.vv v1, v2, v3     // 非法指令异常

// Zve32f: 需要 F 扩展，可以使用向量浮点指令
vfadd.vv v1, v2, v3     // 合法
```

**应用场景：**
- **嵌入式 AI**：Zve32x 足够，无需标量浮点
- **数字信号处理**：Zve32f 提供单精度浮点
- **高性能计算**：V 完整扩展提供双精度浮点

## 6. 指令编码变化

### 6.1 主要编码差异

#### 6.1.1 vsetvli 指令编码

**v0.7.1 vs v1.0：**

| 字段 | v0.7.1 | v1.0 | 变化 |
|------|--------|------|------|
| vlmul[2:0] | 3 位 | 3 位 | 支持分数 LMUL |
| vsew[2:0] | 3 位 | 3 位 | 相同 |
| vediv[1:0] | 2 位 | **移除** | v1.0 不支持 |
| vma | 不存在 | 1 位 | 新增 |
| vta | 不存在 | 1 位 | 新增 |

**编码示例：**

```
// v0.7.1: vsetvli t0, a0, e32, m1
// 编码：[31:26]=OP-V, [25]=1, [24:20]=rd, [19:15]=rs1,

#### 10.1.1 v0.7.1 实现

void vector_add_v071(float *a, float *b, float *c, int n) {
    // 设置向量参数
    vsetvli t0, zero, e32, m1
    
    // 主循环
    for (int i = 0; i < n; ) {
        vsetvli t0, a0, e32, m1  // 设置 VL
        vle32.v v1, (a)          // 加载 a
        vle32.v v2, (b)          // 加载 b
        vfadd.vv v3, v1, v2      // 向量加法
        vse32.v v3, (c)          // 存储 c
        
        // 更新指针
        slli t1, t0, 2           // t1 = VL * 4
        add a, a, t1
        add b, b, t1
        add c, c, t1
        sub a0, a0, t0
    }
}
```

void vector_add_v10(float *a, float *b, float *c, int n) {

#### 10.2.1 数据结构

#### 10.2.2 v0.7.1 实现

```c
void update_particles_v071(struct Particle *p, int n, float dt) {
    for (int i = 0; i < n; ) {
        vsetvli t0, a0, e32, m1
        
        // 加载 x 坐标（需要计算偏移）
        add t1, a0, 0
        vle32.v v1, (t1)
        
        // 加载 vx（需要计算偏移）
        add t1, a0, 12  // 偏移 12 字节
        vle32.v v2, (t1)
        
        // 更新 x
        vfmul.vf v3, v2, dt
        vfadd.vv v1, v1, v3
        
        // 存储 x
        add t1, a0, 0
        vse32.v v1, (t1)
        
        // ... 重复 y, z
        
        // 更新指针（每个粒子 24 字节）
        slli t1, t0, 3   // t1 = VL * 8
        add t1, t1, t1   // t1 = VL * 16
        add t1, t1, t0   // t1 = VL * 24
        add a0, a0, t1
        sub a1, a1, t0
    }
}
```

#### 10.2.3 v1.0 实现（使用索引加载）

```c
void update_particles_v10(struct Particle *p, int n, float dt) {
    // 生成索引：0, 1, 2, ...
    vid.v v0
    
    for (int i = 0; i < n; ) {
        vsetvli t0, a0, e32, m1, ta=1, ma=1
        
        // 计算字节偏移：i * 24
        vmul.vx v1, v0, 24
        
        // 加载 x（使用 32 位索引）
        vluxei32.v v2, (a0), v1
        
        // 计算速度偏移：i * 24 + 12
        vadd.vx v3, v1, 12
        vluxei32.v v4, (a0), v3
        
        // 更新 x
        vfmul.vf v5, v4, dt
        vfadd.vv v2, v2, v5
        
        // 存储 x
        vsuxei32.v v2, (a0), v1
        
        // ... 重复 y, z
        
        // 更新索引
        vadd.vx v0, v0, t0
        sub a1, a1, t0
    }
}
```
